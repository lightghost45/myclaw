# 自动触发记忆

实现完整的自动记忆循环：捕获（Capture）+ 召回（Recall）。

## 工作流程

```
用户消息 → [Recall] 搜索记忆 → 注入上下文 → 生成回复 → [Capture] 存储新记忆
```

## 一、Auto-Recall（自动召回）

每次收到用户消息时，在生成回复**之前**执行：

### 执行步骤

1. **构建搜索查询**
   - 提取用户消息中的关键主题/实体
   - 可以直接使用用户消息作为查询

2. **调用 memory_search**
   ```typescript
   {
     "name": "memory_search",
     "parameters": {
       "query": "<用户消息或提取的关键主题>",
       "maxResults": 5,
       "minScore": 0.3
     }
   }
   ```

3. **注入上下文**
   - 如果找到相关记忆，在 system prompt 或对话开头注入：
   ```
   相关记忆：
   - <记忆1>
   - <记忆2>
   ```

4. **记录召回状态**
   - 记录本次召回了多少条记忆
   - 可选：告知用户"已加载X条相关记忆"

### 触发条件

- 每次用户发送消息时
- 如果当前对话已有上下文（不是全新对话）

## 二、Auto-Capture（自动捕获）

每次生成回复**之后**执行：

### 执行步骤

1. **分析对话内容**
   - 检查用户消息和回复中是否有值得记忆的信息
   
2. **触发条件判断**

   | 条件类型 | 示例 | 记忆位置 |
   |---------|------|---------|
   | 明确偏好 | "我喜欢..."、"以后都用..." | MEMORY.md |
   | 关键决定 | "那就用..."、"决定是..." | MEMORY.md |
   | 重要事件 | "第一次..."、"纪念日" | memory/YYYY-MM-DD.md |
   | 承诺/约定 | "记住..."、"别忘了..." | memory/YYYY-MM-DD.md |
   | 重复话题 | 同一话题出现3次以上 | MEMORY.md |
   | 情绪变化 | 明显的心情/状态变化 | memory/YYYY-MM-DD.md |

3. **写入记忆**
   - 使用 write/edit 工具
   - 短期信息 → `memory/YYYY-MM-DD.md`
   - 长期信息 → `MEMORY.md`

4. **去重检查**
   - 写入前搜索是否已有相似记录
   - 相似度 > 80% 则不重复写入

### 写入格式

```markdown
## 时间 - 事件类型

- 关键信息点
- 另一要点
```

## 三、配置选项

可通过环境变量或配置调整行为：

| 变量 | 默认值 | 描述 |
|------|-------|------|
| `MEMORY_RECALL_ENABLED` | true | 是否启用自动召回 |
| `MEMORY_CAPTURE_ENABLED` | true | 是否启用自动捕获 |
| `MEMORY_RECALL_TOP_K` | 5 | 召回时返回的最大条数 |
| `MEMORY_RECALL_THRESHOLD` | 0.3 | 召回时的最小相似度阈值 |

## 四、与 OpenClaw 内置记忆的关系

| 功能 | 本 Skill | OpenClaw 内置 |
|------|---------|--------------|
| 召回 | 每次对话前自动 | 需手动调用 memory_search |
| 捕获 | 每次对话后自动判断 | 仅预压缩/会话结束触发 |
| 存储 | Markdown 文件 | Markdown 文件 |
| 检索 | memory_search | memory_search |

本 Skill 增强的是**触发时机**的自动化，不替代底层存储和检索。

## 五、示例流程

**场景**：用户说"我咖啡只喝拿铁，不要卡布奇诺了"

### Recall（对话前）
- 搜索：用户偏好、饮食相关记忆
- 结果：无相关记录
- 动作：正常生成回复

### Capture（对话后）
- 分析：用户表达明确偏好
- 触发：写入 MEMORY.md
- 结果：
```markdown
## 偏好更新
- 咖啡偏好：只喝拿铁，不再点卡布奇诺（2026-03-26）
```

**下次对话**：用户说"帮我买杯咖啡"
- Recall 搜索"咖啡" → 找到上述偏好
- 注入上下文："用户只喝拿铁"
- 生成的回复会更贴合用户偏好
