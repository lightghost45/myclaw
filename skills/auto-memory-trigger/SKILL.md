# 自动触发记忆

你需要在每次对话中**主动识别**需要记忆的内容，并在适当时机写入记忆文件。

## 触发条件

当对话中出现以下情况时，自动触发记忆写入：

### 1. 明确偏好表达
- 用户说"我喜欢..."、"我偏好..."、"我习惯..."
- 用户说"以后都用..."、"下次记得..."

### 2. 关键决定
- 用户做出决定："那就用..."、"决定是..."、"选..."
- 用户改变主意或更新决定

### 3. 重要事件
- 第一次做某事（"第一次..."）
- 纪念日、里程碑
- 重大变化、转折点

### 4. 承诺/约定
- 用户说"记住..."、"别忘了..."
- 约定的时间、任务、待办

### 5. 重复提及
- 同一话题出现 3 次以上，可能是重要事项
- 用户反复强调的内容

### 6. 情绪/状态变化
- 用户表达焦虑、困扰、开心等强烈情绪
- 状态明显变化

## 写入规则

### 写入位置

| 内容类型 | 目标位置 |
|---------|---------|
| 短期/日常信息 | memory/YYYY-MM-DD.md |
| 长期/重要信息 | MEMORY.md |

### 写入格式

```markdown
## 时间 - 事件描述

- 关键信息点
- 另一要点
```

### 去重检查

写入前，检查是否已存在相似记录：
1. 搜索 MEMORY.md 和 memory/*.md
2. 如果内容高度相似（>80%），则不重复写入
3. 可以更新或补充，而不是覆盖

## 执行流程

1. **监听**：对话中持续关注上述触发条件
2. **判断**：识别到触发条件时，评估是否值得记录
3. **写入**：选择合适位置，写入记忆文件
4. **确认**：简单告知用户已记录（可选，避免打扰）

## 示例

**用户说**："我以后咖啡只喝拿铁，不要再点卡布奇诺了"

→ 写入 MEMORY.md：
```markdown
## 偏好更新

- 咖啡偏好：只喝拿铁，不再点卡布奇诺
```

**用户说**："下周三是我们的纪念日"

→ 写入 memory/YYYY-MM-DD.md：
```markdown
## 重要提醒

- 下周三是纪念日（提前记录）
```
